# 클린 코드
## 감싸기(wrapper) 클래스
만약 외부 API를 사용하는데, 우리 서비스에서 사용하지 않는 다양한 예외가 존재한다면, 감싸기 클래스로 외부 라이브러리와 프로그램 사이에 의존성을 크게 줄일 수 있다.  

```java
public class 감싸기클래스 {
    private 외부라이브러리 외라;

    public 감싸기클래스(int i) {
        외라 = new 외부라이브러리(i);
    }

    public void open() {
        try {
            외라.open();
        } catch(외부예외1 e) {
            throw 커스텀예외(e);
        } catch(외부예외2 e) {
            throw 커스텀에외(e);
        }
    }
}
```
이렇게 하면 나는 외부라이브러리의 다양한 예외에 대해서 프로젝트 내에서 핸들링할 필요 없이, 커스텀예외 하나만 핸들링하면 된다.
